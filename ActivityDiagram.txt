Diagram Aktivitas untuk Website Thrift Shop - Proses Pembelian

Diagram aktivitas ini menggunakan swimlanes untuk memisahkan aktivitas berdasarkan aktor: Pembeli (Pelanggan), Sistem, dan Admin. Proses pembelian utamanya melibatkan Pembeli dan Sistem, sedangkan Admin terlibat dalam pengelolaan order setelah pembelian.

**Aktivitas berdasarkan Aktor:**

**Pembeli:**
- Mulai: Mengakses website
- Jelajahi Produk: Melihat produk di halaman Home atau Produk
- Lihat Detail Produk (opsional): Klik produk untuk melihat detail
- Putuskan Tambah ke Keranjang: Jika tertarik, lanjut tambah ke keranjang
- Lanjut Belanja atau Checkout: Bisa tambah item lagi atau lanjut ke checkout
- Lihat Keranjang: Navigasi ke halaman Keranjang
- Update Keranjang (opsional): Ubah jumlah atau hapus item
- Lanjut ke Checkout: Klik tombol checkout
- Masukkan Detail Pelanggan: Isi nama, telepon, alamat
- Konfirmasi Order: Review ringkasan order dan total
- Upload Bukti Pembayaran: Upload bukti pembayaran
- Submit Pembayaran: Kirim bukti ke server

**Sistem:**
- Periksa Status Login: Apakah pelanggan sudah login?
- Prompt Login/Daftar: Jika belum login, tampilkan prompt
- Tambah Produk ke Keranjang: Tambah produk ke state keranjang
- Buat Order: Buat order dengan status 'pending'
- Redirect ke Halaman Pembayaran: Navigasi ke halaman pembayaran
- Update Status Order: Update status order ke 'paid'
- Tampilkan Konfirmasi: Tampilkan konfirmasi order

**Admin:**
- Admin tidak terlibat langsung dalam proses pembelian, tetapi dapat mengelola order setelah pembelian (misalnya update status order dari 'paid' ke 'shipped' atau 'completed').

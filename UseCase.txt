Use Case Diagram for Thrift Shop Website

Actors:
1. Customer (User with role 'user')
2. Admin (User with role 'admin')

Use Cases for Customer:
1. Register Account
   - Description: Customer creates a new account with username and password.
   - Precondition: Customer is not logged in.
   - Postcondition: Account created, customer logged in.

2. Login
   - Description: Customer logs in with username and password.
   - Precondition: Customer has an account.
   - Postcondition: Customer is authenticated and logged in.

3. Browse Products
   - Description: Customer views list of available products.
   - Precondition: None.
   - Postcondition: Products displayed.

4. View Product Details
   - Description: Customer views detailed information of a specific product.
   - Precondition: Product exists.
   - Postcondition: Product details shown.

5. Add to Cart
   - Description: Customer adds a product to their shopping cart.
   - Precondition: Customer is logged in, product available.
   - Postcondition: Product added to cart.

6. View Cart
   - Description: Customer views items in their cart.
   - Precondition: Customer is logged in.
   - Postcondition: Cart contents displayed.

7. Update Cart Quantity
   - Description: Customer changes quantity of items in cart.
   - Precondition: Items in cart.
   - Postcondition: Cart updated.

8. Remove from Cart
   - Description: Customer removes an item from cart.
   - Precondition: Item in cart.
   - Postcondition: Item removed.

9. Add to Wishlist
   - Description: Customer adds a product to their wishlist.
   - Precondition: Customer logged in, product exists.
   - Postcondition: Product added to wishlist.

10. View Wishlist
    - Description: Customer views their wishlist.
    - Precondition: Customer logged in.
    - Postcondition: Wishlist displayed.

11. Remove from Wishlist
    - Description: Customer removes a product from wishlist.
    - Precondition: Product in wishlist.
    - Postcondition: Product removed.

12. Checkout
    - Description: Customer initiates checkout process for items in cart.
    - Precondition: Items in cart, customer logged in.
    - Postcondition: Order created, redirected to payment.

13. Make Payment
    - Description: Customer uploads payment proof for the order.
    - Precondition: Order pending.
    - Postcondition: Payment proof uploaded, order status updated.

14. View Order History
    - Description: Customer views their past orders.
    - Precondition: Customer logged in.
    - Postcondition: Orders displayed.

15. View Profile
    - Description: Customer views their profile information.
    - Precondition: Customer logged in.
    - Postcondition: Profile shown.

16. Edit Profile
    - Description: Customer updates their profile.
    - Precondition: Customer logged in.
    - Postcondition: Profile updated.

17. Logout
    - Description: Customer logs out.
    - Precondition: Customer logged in.
    - Postcondition: Session ended.

Use Cases for Admin:
1. Login (same as customer, but role 'admin')

2. View Dashboard
   - Description: Admin views overview of site statistics.
   - Precondition: Admin logged in.
   - Postcondition: Dashboard displayed.

3. Manage Products
   - Includes: Create Product, Update Product, Delete Product, View Product List
   - Description: Admin performs CRUD operations on products.
   - Precondition: Admin logged in.
   - Postcondition: Products updated.

4. Manage Orders
   - Includes: View Order List, View Order Details, Update Order Status
   - Description: Admin views and updates order statuses.
   - Precondition: Admin logged in.
   - Postcondition: Orders managed.

5. Manage Users
   - Includes: View User List
   - Description: Admin views list of users.
   - Precondition: Admin logged in.
   - Postcondition: Users displayed.

6. Logout (same as customer)

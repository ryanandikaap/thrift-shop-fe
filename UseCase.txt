Diagram Kasus Penggunaan untuk Website Thrift Shop

Aktor:
1. Pelanggan (User dengan role 'user')
2. Admin (User dengan role 'admin')

Kasus Penggunaan untuk Pelanggan:
1. Daftar Akun
   - Deskripsi: Pelanggan membuat akun baru dengan username dan password.
   - Prasyarat: Pelanggan belum login.
   - Pasca kondisi: Akun dibuat, pelanggan login.

2. Login
   - Deskripsi: Pelanggan login dengan username dan password.
   - Prasyarat: Pelanggan memiliki akun.
   - Pasca kondisi: Pelanggan terautentikasi dan login.

3. Jelajahi Produk
   - Deskripsi: Pelanggan melihat daftar produk yang tersedia.
   - Prasyarat: Tidak ada.
   - Pasca kondisi: Produk ditampilkan.

4. Lihat Detail Produk
   - Deskripsi: Pelanggan melihat informasi detail produk tertentu.
   - Prasyarat: Produk ada.
   - Pasca kondisi: Detail produk ditampilkan.

5. Tambah ke Keranjang
   - Deskripsi: Pelanggan menambah produk ke keranjang belanja.
   - Prasyarat: Pelanggan login, produk tersedia.
   - Pasca kondisi: Produk ditambah ke keranjang.

6. Lihat Keranjang
   - Deskripsi: Pelanggan melihat item di keranjang.
   - Prasyarat: Pelanggan login.
   - Pasca kondisi: Isi keranjang ditampilkan.

7. Update Jumlah Keranjang
   - Deskripsi: Pelanggan mengubah jumlah item di keranjang.
   - Prasyarat: Item di keranjang.
   - Pasca kondisi: Keranjang diupdate.

8. Hapus dari Keranjang
   - Deskripsi: Pelanggan menghapus item dari keranjang.
   - Prasyarat: Item di keranjang.
   - Pasca kondisi: Item dihapus.

9. Tambah ke Wishlist
   - Deskripsi: Pelanggan menambah produk ke wishlist.
   - Prasyarat: Pelanggan login, produk ada.
   - Pasca kondisi: Produk ditambah ke wishlist.

10. Lihat Wishlist
    - Deskripsi: Pelanggan melihat wishlist mereka.
    - Prasyarat: Pelanggan login.
    - Pasca kondisi: Wishlist ditampilkan.

11. Hapus dari Wishlist
    - Deskripsi: Pelanggan menghapus produk dari wishlist.
    - Prasyarat: Produk di wishlist.
    - Pasca kondisi: Produk dihapus.

12. Checkout
    - Deskripsi: Pelanggan memulai proses checkout untuk item di keranjang.
    - Prasyarat: Item di keranjang, pelanggan login.
    - Pasca kondisi: Order dibuat, diarahkan ke pembayaran.

13. Lakukan Pembayaran
    - Deskripsi: Pelanggan upload bukti pembayaran untuk order.
    - Prasyarat: Order pending.
    - Pasca kondisi: Bukti pembayaran diupload, status order diupdate.

14. Lihat Riwayat Order
    - Deskripsi: Pelanggan melihat order sebelumnya.
    - Prasyarat: Pelanggan login.
    - Pasca kondisi: Order ditampilkan.

15. Lihat Profil
    - Deskripsi: Pelanggan melihat informasi profil.
    - Prasyarat: Pelanggan login.
    - Pasca kondisi: Profil ditampilkan.

16. Edit Profil
    - Deskripsi: Pelanggan update profil.
    - Prasyarat: Pelanggan login.
    - Pasca kondisi: Profil diupdate.

17. Logout
    - Deskripsi: Pelanggan logout.
    - Prasyarat: Pelanggan login.
    - Pasca kondisi: Sesi berakhir.

Kasus Penggunaan untuk Admin:
1. Login (sama dengan pelanggan, tapi role 'admin')

2. Lihat Dashboard
   - Deskripsi: Admin melihat overview statistik situs.
   - Prasyarat: Admin login.
   - Pasca kondisi: Dashboard ditampilkan.

3. Kelola Produk
   - Termasuk: Buat Produk, Update Produk, Hapus Produk, Lihat Daftar Produk
   - Deskripsi: Admin melakukan operasi CRUD pada produk.
   - Prasyarat: Admin login.
   - Pasca kondisi: Produk diupdate.

4. Kelola Order
   - Termasuk: Lihat Daftar Order, Lihat Detail Order, Update Status Order
   - Deskripsi: Admin melihat dan update status order.
   - Prasyarat: Admin login.
   - Pasca kondisi: Order dikelola.

5. Kelola User
   - Termasuk: Lihat Daftar User
   - Deskripsi: Admin melihat daftar user.
   - Prasyarat: Admin login.
   - Pasca kondisi: User ditampilkan.

6. Logout (sama dengan pelanggan)
